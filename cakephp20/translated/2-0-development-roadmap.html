以下はCakePHPの機能の「やっつけリスト」（訳注："hit list"）になります。

優先順位リスト

  • E_STRICTへの準拠
  • 適切なアクセス修飾子の適応とCake自身のアクセス修飾子違反の修正
  • PHP4ハック（訳注：PHP4にないPHP5の機能・振る舞いのためのハック）の削除
  • 非推奨のクラスと機能の削除
  • FileクラスとFolderクラスのSplFileObjectを用いた置き換え
  • Fileクラスの非推奨化
  • Folderクラスの非推奨化
  • 可能なところはSplを使う
  • XMLの悲惨な状況の修正。SimpleXmlが必要なほとんどのものを提供することになるでしょう。
  • モデル、ヘルパー、コンポーネントのLazy loadingと初期化
  • 可能なところは例外処理、インターフェース、抽象クラスを使う
  • Clean up the global namespace. ClassRegistry is heavily abused right now. It basically makes N global symbols. Global symbols are evil, and removing them improves decoupling.
  • Remove magic that thwarts 'What you type is what you get' #858 is a perfect example of this.

Unsorted

  • フレームワークのより良い分離
  • ヘルパのエイリアス
    • 例: $helpers = array('Html' => 'MyCustomHtml');
  • ビヘイビアのエイリアス
    • 例: $actsAs = array('Containable' => 'MyCustomContainable');
  • コンポーネントのエイリアス
    • 例: $components = array('Html' => 'MyCustomHtml');
  • モデルの書き直し
    • オブジェクトを返り値とする（訳注：find()の戻り値、例：$data->user_nameや$data->Group->idなどとなる？）
    • DataMapperパターンに近いモデリング
  • ビヘイビア
    • ClassRegistryから削除
  • DboSource
    • 可視性の問題・アクセスの修正
  • Containableビヘイビア
    • 可視性の問題・アクセスの修正
