The below is a "hit list" of features for the CakePHP.

Prioritized List

  • Compliance with E_STRICT
  • Apply proper visibility and fix visibility violations in cake itself.
  • Remove PHP4 hacks.
  • Remove deprecated classes and features.
  • Replace file and folder class with SplFileObject
  • Deprecate File class
  • Deprecate Folder class
  • use Spl where possible
  • Fix XML, its horrible. SimpleXml will provide almost everything we need.
  • Lazy loading and construction of models, helpers and components.
  • Use exceptions, interfaces, and abstract classes where possible and logical.
  • Clean up the global namespace. ClassRegistry is heavily abused right now. It basically makes N global symbols. Global symbols are evil, and removing them improves decoupling.
  • Remove magic that thwarts 'What you type is what you get' #858 is a perfect example of this.

Unsorted

  • Further decouple the framework
  • Aliases for Helpers
    • Example: $helpers = array('Html' => 'MyCustomHtml');
  • Aliases for Behaviors
    • Example: $actsAs = array('Containable' => 'MyCustomContainable');
  • Aliases for Components
    • Example: $components = array('Html' => 'MyCustomHtml');
  • Model rewrite
    • Returning objects
    • Modelling closer to DataMapper pattern
  • Behaviors
    • Remove them from the ClassRegistry
  • DboSource
    • Fix visibility issuess / accesses
  • Containable Behavior
    • Fix visibility issuess / accesses
