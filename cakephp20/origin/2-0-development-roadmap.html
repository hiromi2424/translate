The below is a "hit list" of features for the CakePHP.

Prioritized List

  • Compliance with E_STRICT
  • Apply proper visibility and fix visibility violations in cake itself.
  • Remove PHP4 hacks.
  • Remove deprecated classes and features.
  • Replace file and folder class with SplFileObject
  • Deprecate File class
  • Deprecate Folder class
  • use Spl where possible
  • Fix XML, its horrible. SimpleXml will provide almost everything we need.
  • Lazy loading and construction of models, helpers and components.
  • Use exceptions, interfaces, and abstract classes where possible and logical.
  • Clean up the global namespace. ClassRegistry is heavily abused right now. It basically makes N global symbols. Global symbols are evil, and removing them improves decoupling.
  • Remove magic that thwarts 'What you type is what you get' #858 is a perfect example of this.

Components

  • Separate CSRF and form tampering features in SecurityComponent
  • Clean up how session configuration works, and make session configuration more powerful/flexible

  • Extract the case switches inside AuthComponent, to use polymorphism. This will make the component more flexible.

    • Extract out how access checks are done
    • Extract out how identity is verified, allowing things like basic auth, form post, and openid possible.

  • Make EmailComponent features available in other parts of an application, like a shell.

  • Make IniAcl work with AuthComponent, right now it is non-functional. It should work.
  • Extract pagination into a component. Split paginate() into smaller methods that will allow more extensibility.
  • PaginatorComponent should allow behaviors to define paginate() and paginateCount(). (markstory)
  • PaginatorComponent should be able to use querystring parameters for paging. (markstory)

Controller

  • Refactor out Response and Request objects
  • Add escaping to Scaffold views.

View

  • Move callbacks, to allow more control over view vars, and view flow.
  • Allow helpers to be loaded at runtime.

Helpers

  • Add html5 support to FormHelper
  • Create a way to fix the delete as a GET request issue.
  • Decouple CacheHelper from View.
  • PaginatorHelper should be able to use querystring parameters for paging. (markstory)

Console

  • Make Shell option parsing and help text generation simpler.
  • Coloured output.
  • Make help not hell to generate
  • Provide argument/option validation
  • Support various levels of output (verbose, quiet)
  • Add app/console.
  • Rebuild the directory structure to make sense.
  • Move shells out of app/vendors and into app/console.
  • Move shell paths into App. The current shellPaths stuff is not so hot.
  • Create Shell::runShell() to replace the current approach to running other shells, from shells.
  • Bake output should include output escaping.

Libs

  • Drop cakeError and convert errors into exceptions where it makes sense.

Datasources

  • Convert datasources to use PDO where possible.

Unsorted

  • Further decouple the framework

  • Aliases for Helpers. Behaviors, and Components. Aliases would be provided as a 'setting' much like 'enabled' is. e.g $helpers = array('MyHtml' => array('alias' => 'Html')); (jeremyharris)

  • Behaviors

    • Remove them from the ClassRegistry (markstory)
  • DboSource
    • Fix visibility issuess / accesses
  • Containable Behavior
    • Fix visibility issuess / accesses
